# 任务交接

为新的 Agent 生成紧凑、基于事实的续接记录。

## 必须包含

- 用户最新目标，以及覆盖早期要求的后续修正。
- 仓库、分支、工作区状态、相关提交，以及是否已提交或推送。
- 已完成的行为和具体文件路径，不按聊天时间线复述。
- 按依赖顺序列出剩余工作；必要时记录失败命令、错误、端口和外部阻塞。
- 引用已有规格、计划、diff 和产物，不重复粘贴。
- 写明实际执行过的验证及结果，不把未运行检查写成通过。

## 安全

删除 API Key、Token、密码、Cookie、个人标识和无关本地信息。除非用户明确要求，不得指示后续 Agent 覆盖或丢弃未提交改动。

使用固定章节：目标、当前状态、已完成、剩余工作、关键文件、验证、约束与决策、建议 Skill、恢复命令。优先保存到操作系统临时目录；无法写文件时直接返回 Markdown。最后给出一个可以安全恢复工作的准确命令或动作。
