# 前端设计

构建有明确设计判断的界面，而不是拼装常见模板。

## 编码前

1. 阅读现有产品和设计规范；除非明确要求重做，否则保持既有体系。
2. 明确受众、页面的首要任务，以及一个来自产品领域本身的视觉主题。
3. 定义简洁系统：4-6 个颜色、字体角色、间距节奏、布局约束和一个标志性元素。
4. 删除那些可以原样套到任何产品上的选择。

## 实现要求

- 优先实现真正可用的业务页面，不先做功能宣传页。
- 运营工具强调扫描、比较和重复操作；不要滥用营销式大标题和卡片。
- 复用仓库已有组件和图标库。
- 为棋盘、工具栏、网格和控件设置稳定的响应式尺寸，避免内容溢出或重叠。
- 文案使用明确标签、主动命令、可执行错误信息和有方向的空状态。
- 支持键盘焦点、降低动态效果、足够对比度和移动端布局。

## 验证

启动应用，在桌面和移动尺寸检查截图及主要交互，同时覆盖长文本、加载、空、错误和选中状态。修复裁切、重叠、空白画布、布局跳动和不可访问控件后再结束。
