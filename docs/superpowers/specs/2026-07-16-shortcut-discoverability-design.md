# 应用内快捷键展示设计

## 背景

应用已经支持搜索、会话选择、详情查看和会话恢复等键盘操作，但设置页只展示可配置的全局唤起快捷键。用户可以在搜索框旁看到 `Enter` 和恢复会话快捷键，也能从少量按钮提示中发现详情搜索键位，其余操作缺少统一入口。

## 目标

- 在设置页集中展示常用的应用内快捷键。
- 根据当前系统使用 `⌘` 或 `Ctrl`，避免显示无法执行的键位。
- 保留现有全局快捷键配置，不让用户误以为应用内快捷键可以修改。
- 展示内容与实际键盘监听保持一致。

## 不包含的范围

- 不增加快捷键自定义能力。
- 不新增独立帮助弹窗或新的键盘入口。
- 不列出 `PageUp`、`PageDown`、`Home`、`End` 等通用滚动键。
- 不列出 macOS 的隐藏、退出、窗口关闭等系统菜单快捷键。

## 设置页布局

在“设置 → 全局快捷键”中保留现有“打开搜索窗口”选择器，并在其下方增加“应用内快捷键”只读区域。

每一行左侧是操作名称，右侧使用 `<kbd>` 展示按键。按键区域只用于说明，不提供点击、录制或编辑能力。列表沿用设置页现有的间距、边框和文字层级，窄窗口允许按键组合自然换行，不增加横向滚动。

中英文标题分别为“应用内快捷键”和“App shortcuts”。辅助说明分别为“这些快捷键不可自定义。”和“These shortcuts cannot be customized.”

## 展示清单

| 操作 | macOS | Windows / Linux |
|---|---|---|
| 聚焦搜索 | `⌘ K` | `Ctrl K` |
| 执行搜索 | `Enter` | `Enter` |
| 选择会话 | `↑ / ↓` | `↑ / ↓` |
| 打开详情 | `Space` | `Space` |
| 恢复选中会话 | `⌘ Enter` | `Ctrl Enter` |
| 会话内查找 | `⌘ F` | `Ctrl F` |
| 上一个 / 下一个匹配 | `Shift Enter / Enter` | `Shift Enter / Enter` |
| 关闭当前面板或弹窗 | `Esc` | `Esc` |

中文操作名称使用上表文字。英文依次为 “Focus search”、“Search”、“Select session”、“Open details”、“Resume selected session”、“Find in conversation”、“Previous / next match”和“Close current panel or dialog”。

全局唤起键继续由上方选择器展示，不在只读清单中重复。

## 实现边界

- 直接在现有 `SettingsDialog` 的快捷键区域增加静态清单，不新增设置状态或持久化字段。
- 平台差异只影响修饰键显示，继续使用现有 `RUNTIME_PLATFORM` 判断。
- 不为单次渲染提取新的导出函数；测试通过设置页的可观察结构验证展示结果。
- 样式放入现有 renderer 样式表，类名限定在快捷键清单范围内，避免影响搜索框已有的 `.kbd-hint`。

## 测试

- 先增加失败的 renderer 契约测试，要求设置页包含全部八项操作及对应按键。
- 测试 macOS 使用 `⌘`，Windows / Linux 使用 `Ctrl` 的条件分支。
- 测试清单使用 `<kbd>`，且没有输入框、选择器或快捷键录制逻辑。
- 运行相关 renderer 测试、完整测试、类型检查、生产构建和 release note 检查。

## 验收标准

- 用户打开“设置 → 全局快捷键”即可看到完整的应用内快捷键清单。
- macOS 与 Windows / Linux 的修饰键显示正确。
- 清单中的每个操作都与当前代码中的实际键盘行为一致。
- 全局快捷键仍可配置，应用内快捷键明确显示为只读。
- 中英文界面均无缺失文案，窄窗口不出现横向滚动。
