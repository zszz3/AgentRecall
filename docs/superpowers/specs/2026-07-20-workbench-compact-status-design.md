# 工作台三段式用量总览设计

## 背景

当前工作台顶部曾被压缩为单行状态带，Token 构成、来源明细和额度重置时间被移入悬停提示。实际使用中信息层级过于扁平，核心数字拥挤，原本一眼可见的解释信息也消失了。

本次主要调整顶部总览的信息层级和布局，并补足工作台会话行缺失的 Open / Closed 状态。它不改变统计口径、额度数据来源、刷新行为或会话查询规则。

## 目标

- 宽窗口下使用约 142px 高的三段式总览，在紧凑和可读之间保留明确层级。
- 首屏直接保留会话数、消息数、Token、缓存率、Token 构成、来源用量，以及 Codex 和 Claude Code 的 5 小时、7 天剩余额度与重置时间。
- Codex 和 Claude Code 合并进同一张额度卡，内部左右并列并共用一个刷新入口。
- 最右保留一个独立模块位，后续增加工作台内容时不需要重做顶部骨架。
- 保留用量周期切换、统计刷新、额度不可用说明和设置入口。
- 让每条工作台会话都能直接看出当前是 Open 还是 Closed，而不是只能通过活跃动画猜测。

## 布局

顶部使用三张独立表面组成一行：

1. 用量区：周期切换、统计刷新和四项核心指标在第一层；Token 构成条、构成图例、缓存说明和来源 Token 明细在第二层。
2. 模型额度区：卡片顶部提供唯一刷新入口；Codex 和 Claude Code 在卡片内部左右并列，各自展示 5 小时与 7 天窗口。
3. 独立模块位：保持单独的竖向栏位，当前只显示弱化占位，后续内容不会挤占用量或额度区。

宽窗口下三块区域保持同高，默认高度约 142px。中等窗口下先隐藏独立模块占位，保留用量和额度两列；更窄时模型额度换到用量下方，避免文字覆盖和水平滚动。最低支持窗口宽度继续沿用应用现有约束。

## 信息层级

以下信息继续常驻显示：

- 今天、7 天、30 天、全部四个周期。
- 会话数、消息数、Token、缓存率。
- Token 的输入、缓存输入、输出和推理构成及图形比例。
- 各 Agent 来源的 Token 数。
- Codex 与 Claude Code 名称。
- 每个 Agent 的 5 小时与 7 天剩余百分比和轻量进度条。
- 每档额度的重置时间或数据过期状态。
- 用量区刷新入口和模型额度区唯一的共享刷新入口。

## 状态与反馈

- 统计或额度刷新期间，对应的唯一刷新图标继续显示旋转状态，其他区域保持可读。
- 额度不可用时，在该 Agent 区域内显示一行简短原因和设置入口，不恢复为高卡片。
- 刷新成功或失败反馈仅在发生时临时出现，不为反馈预留固定高度。
- 缓存率无有效分母时继续显示 `—`，不显示误导性的 `0%`。

## 会话行状态

- 继续复用现有会话进程检测结果，不新增另一套 Open 判断。
- 在项目与 Agent 来源副信息末尾显示状态；Open 使用绿色状态点和文字，Closed 使用弱化的中性色文字。
- 状态跟随应用语言显示为 `Open / Closed` 或 `打开 / 关闭`。
- 活跃会话原有的会话轨迹动画继续保留，用来突出正在运行的条目；文字状态用于提供明确语义，两者不互相替代。
- 状态放入现有副信息行，不新增表格列，也不增加会话行高度。

## Provider Tab

- 左侧主导航在工作台、会话和 Skills 之后增加 `Provider` Tab，使用现有 Key 图标。
- 点击后在主工作区直接显示当前 API 配置的全部内容，包括 Codex、Claude Code、AI 摘要与搜索三个内部页签。
- Provider 页面移除弹窗遮罩、关闭按钮和弹窗尺寸限制；页面主体独立滚动，保存与写入配置操作固定在页面底部。
- 会话页工具栏不再重复显示 API 配置按钮。设置页中已有的 API 配置入口改为关闭设置并切换到 Provider Tab。
- 所有草稿、模型探测、API Key 显隐、保存到应用和写入 Agent 配置的行为保持不变。

## 页面标题栏

- 工作台、会话、Skills 和 Provider 在主内容顶部使用相同高度、分隔线和文字层级的标题栏，替代 macOS 窗口下贯穿页面的纯留白。
- 工作台标题保持“工作台 / Workbench”，副标题固定为 `One for all`。
- 会话页说明其搜索、筛选和继续本地或远程会话的用途；Skills 页说明本地管理与跨设备同步；Provider 保留现有说明。
- 会话页移除重复的设置按钮，更新提示和设置入口统一由左侧主导航承担。

## 实现边界

- 复用现有统计和额度数据，不增加 IPC、数据库字段或后台请求。
- 直接调整现有工作台组件，不建立仅用于转发属性的包装组件。
- 恢复常驻的 Token 构成条和来源明细，并移除依赖悬停才能理解统计的提示层。
- 两个模型额度继续复用同一渲染逻辑，但刷新按钮由父级额度卡统一持有。
- Provider 直接复用并改造现有唯一的 API 配置组件，不同时保留页面和弹窗两套外壳。
- 不改变工作台会话搜索、会话排序或 Workflow 预留区域。

## 验收标准

- 宽窗口下顶部总览为三段式单行，默认高度约 142px。
- 四项统计、Token 构成、来源 Token，以及两种 Agent 的两档额度和重置时间无需展开即可看见。
- Codex 和 Claude Code 在同一张卡内左右并列，整个额度卡只有一个刷新按钮。
- 最右存在独立模块位，中等窗口下可优先隐藏而不影响主要信息。
- 刷新、加载、额度不可用和设置跳转继续工作。
- 中等窗口不发生文字覆盖或水平溢出。
- 每条工作台会话均显示与现有进程检测一致的 Open / Closed 状态，且行高不增加。
- 左侧可直接进入 Provider 页面；原 API 配置功能完整，且会话工具栏不再出现重复入口。
- 四个主页面均有一致的标题栏，会话页工具栏不再重复显示设置按钮。
- 现有统计与额度测试通过，并新增紧凑结构和响应式布局契约测试。
