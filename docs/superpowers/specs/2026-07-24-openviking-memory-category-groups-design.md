# OpenViking 记忆分类折叠设计

## 背景

Memory 页面已经可以列出一个受管理目录的全部 OpenViking 记忆，但当前把不同类型的记忆平铺在同一个长列表中。OpenViking 已通过记忆 URI 的目录层级提供类型信息，页面不应丢失这层结构。

## 目标

- 按 OpenViking 记忆类型对现有列表分组。
- 每个分组可以独立展开或折叠，并显示记忆数量。
- 首次进入、切换目录以及获得一批新结果时，所有非空分组默认展开。
- 搜索结果继续使用同一套分类展示。
- 不修改 OpenViking 的存储结构、搜索接口或记忆正文。

## 非目标

- 不增加服务端分类接口。
- 不允许用户创建、重命名或删除 OpenViking 内置分类。
- 不持久化折叠状态。
- 不增加跨分类排序、拖拽或批量操作。
- 不改变单条记忆的查看、编辑或删除语义。

## 分类映射

页面优先根据记忆 URI 中 `viking://user/memories/` 后的首层目录归类；URI 没有分类层级时，再使用 `OpenVikingMemoryItem.source`。这能避免语义搜索结果把来源会话 ID 误当成分类。分类使用稳定顺序：

| OpenViking 来源 | 中文显示 | 英文显示 |
| --- | --- | --- |
| `identity.md`, `soul.md` | 身份 | Identity |
| `preferences` | 偏好 | Preferences |
| `cases` | 案例 | Cases |
| `experiences` | 经验 | Experiences |
| `events` | 事件 | Events |
| `trajectories` | 轨迹 | Trajectories |
| `manual` 或没有 ID 的新记忆 | 手动 | Manual |
| 其他或缺失来源 | 其他 | Other |

分类映射只影响展示，不重写记忆的 `source`、URI 或正文。空分类不显示。

## 交互设计

结果栏由一组垂直排列的分类区块组成。每个区块标题包含：

- 展开或折叠箭头；
- 本地化分类名称；
- 当前结果数量。

分类标题是完整宽度按钮。点击标题只切换当前分类，不影响其他分类。全部分类首次显示时默认展开；切换受管理目录或执行一次新搜索后，当前非空分类重新全部展开。

展开后，分类内继续使用现有记忆按钮样式。分类内顺序保持网关返回顺序，不在前端二次排序。折叠分类不会清除当前搜索、选中记忆或详情正文；如果已选中的记忆所在分类被折叠，详情区继续显示该记忆。

列表仍使用已有的独立滚动容器。分类标题随内容滚动，不做吸顶，避免多个展开分类占用额外固定高度。

## 数据流

1. 页面通过现有 `searchOpenVikingMemories` 获得扁平 `OpenVikingMemoryItem[]`。
2. Renderer 按固定分类映射把结果派生成有序分组。
3. Renderer 只为非空分组生成分类标题和记忆列表。
4. 折叠状态保存在当前页面组件内，以分类键为索引。
5. `results` 或 `workspaceId` 发生批次变化时，将当前非空分类设置为展开。

分类是纯展示派生状态，不进入 preload、IPC、主进程或数据库。

## 搜索、加载与空状态

- 加载既有记忆时继续显示当前加载状态。
- 搜索结果按同一分类映射分组。
- 搜索无结果时继续显示现有空状态，不渲染空分类标题。
- 某个分类没有搜索命中时不显示该分类。
- 新建手动记忆仍直接进入详情编辑；保存并重新加载后归入“手动”。

## 响应式布局

桌面宽度维持现有“分类结果栏 + 详情栏”双栏结构。窄窗口仍使用现有单栏断点和结果栏最大高度；分类标题和数量必须保持可见，长标题使用省略号，不能把数量挤出可视区。

## 可访问性

- 分类标题使用原生 `button`。
- 标题设置 `aria-expanded`。
- 标题包含本地化分类名称和数量，不能只依赖图标表达状态。
- 展开箭头带 `aria-hidden`，避免重复朗读。
- 键盘焦点样式沿用现有按钮焦点规则。

## 测试策略

- 单元测试分类映射、稳定顺序、数量和未知来源兜底。
- Renderer 契约测试验证分类标题、`aria-expanded` 与折叠状态接线。
- CSS 契约测试验证分组仍位于现有可滚动结果栏内。
- 手工验证 127 条现有记忆全部展开时可滚动到底部，折叠单个分类后其他分类不受影响，搜索结果仍按分类显示。

## 验收标准

- Memory 页面不再展示一个无结构的扁平长列表。
- 所有非空分类首次进入时均展开，并显示准确数量。
- 用户可以独立折叠和展开任意分类。
- 切换目录和执行搜索后，新的非空分类全部展开。
- 127 条现有记忆仍全部可访问，深层 `events` 记忆不会消失。
- 记忆的搜索、查看、手动创建和删除行为保持不变。
