# AgentRecall 2.0 工作台首页设计

## 背景

AgentRecall 当前首页同时承担会话搜索、筛选、统计、额度、导航和工具入口，首屏信息密度较高，用户启动应用后不能快速回到正在进行或刚刚中断的会话。

AgentRecall 2.0 的首页改为“工作台”。它面向同时使用多个本地 Agent 的开发者，唯一首要任务是帮助用户看清当前状态，并尽快继续工作。完整会话管理和 Skills 管理仍然保留，但进入各自的功能页。

## 目标

- 启动后优先展示 Agent 用量、额度和可继续的会话，并为后续 Workflow 迁移预留稳定位置。
- 活跃会话排在最近会话之前，并通过一个明确的 `Resume` 操作继续工作。
- 现有搜索、会话筛选、详情、远程同步、恢复和 Skills 能力不丢失。
- 首屏先使用已索引的数据渲染，后台刷新不阻塞浏览和操作。
- 将当前宽筛选侧栏替换为稳定的左侧纵向功能导航。
- 延续现有明暗主题和视觉语言，不把首页做成通用数据大屏。

## 不包含的范围

- 本阶段不迁移 multi-agent-chat 的 Chat、Task、Team、Schedule、Workflow、Runtime 配置和 Evaluation 等模块。工作台中的 Workflow 区域只显示空状态，不读取或伪造流程数据，也不提前定义迁移接口。
- 不增加新的 Token 采集字段、Skill 排名算法或会话收藏规则；缓存率只由已有输入与缓存输入数据计算，工作台暂不展示常用 Skills。
- 不提供拖拽、缩放或自定义工作台模块的能力。
- 不改变现有会话详情、恢复路由、活跃状态判断或远程同步语义。
- 不修改项目知识分析功能。

## 信息架构

应用使用固定的左侧纵向 Tab：

- 工作台
- 会话
- Skills
- 设置（固定在底部）

首版不显示 Chat 入口。后续功能只有在完成迁移并可使用后才加入导航，不预先展示禁用入口。

当前侧栏中的环境、项目、标签、来源和视图筛选全部移动到“会话”页内部。AI 找会话、远程会话和会话相关操作继续属于“会话”页；API 配置和应用设置统一从“设置”进入。Skills 从弹窗提升为独立 Tab。会话详情仍以现有详情面板打开。

不设置占据内容高度的全局顶部状态栏。左侧品牌入口使用 README 中现有的 AgentRecall 紫色机器人标志；索引刷新收进左侧导航底部的图标按钮，正常状态不显示“索引已就绪”和上次刷新时间。手动刷新、刷新失败等短暂状态使用现有轻量提示反馈，不长期占用首屏空间。

完整搜索属于“会话”Tab，同时工作台会话模块提供紧凑搜索入口：

- 搜索框位于会话标题和筛选条件之间。
- 工作台搜索框位于会话卡片标题下方，输入时不查询，按 Enter 后才提交并记录最近搜索。
- 工作台搜索结果直接显示在会话卡片中，最多 10 条；清空搜索后恢复默认的活跃优先列表。
- 启动时不自动进入“会话”、不自动聚焦，也不自动展开最近搜索。
- `⌘ K` / `Ctrl K` 和应用菜单触发搜索时，先切换到“会话”Tab，再聚焦搜索框。
- 按 Enter 提交并在当前“会话”Tab 展示结果；现有搜索历史行为保持不变。

### 会话页范围与筛选

“会话”页完整承接当前首页的会话管理能力，不做成只有搜索框的简化列表。筛选分为两个层级：

- 第一层是会话范围，按“位置 → 项目 → Agent”组织。位置可选本机和远程、仅这台电脑、全部远程或某个具体远程环境；项目列表随位置范围变化；Agent 保留全部来源和各来源选项。
- 第二层是状态、时间、标签和视图。状态包含全部、Open、Closed；时间保留全部、今天、7 天和 30 天；视图保留全部、收藏、置顶和隐藏。排序不再作为常驻控件暴露，会话查询统一使用智能排序：有关键词时综合相关性与时间，无关键词时按最近活动排列。

结果行明确展示项目、位置、Agent 来源和最后活动时间。远程会话使用远程环境名称标识，本机会话显示“本机”，避免仅依赖路径猜测会话来自哪里。

位置和项目筛选必须进入数据库查询层，不能只在 renderer 隐藏结果，确保分页总数、统计和“加载更多”正确。“全部远程”覆盖所有启用的 SSH 环境；云端会话归档仍由现有远程会话入口管理，不与 SSH 位置筛选混为一类。

## 工作台布局

```text
┌──────────┬──────────────────────────────────────────────────────┐
│ 工作台   │ 用量统计             Codex 额度       Claude 额度   │
│ 会话     ├──────────────────────────────────────────────────────┤
│ Skills   │ 会话（50%）                  │ Workflow（50%）        │
│          │   搜索                       │ 暂未迁移的空状态       │
│ 刷新     │   活跃优先的 10 条滚动列表   │                        │
│ 设置     │                              │                        │
└──────────┴──────────────────────────────────────────────────────┘
```

工作台由顶部状态总览和中部双栏工作区组成。中部的会话卡片与 Workflow 预留卡片各占 50%，两者是有间距的独立模块。工作台暂不展示 Skills，完整能力只出现在 Skills Tab。

工作台不额外显示“工作台”页面标题或介绍文案。左侧选中状态已经表达当前位置，内容区进入后直接从顶部状态总览开始。

## 顶部状态总览

状态总览是一块连续表面，通过细分隔线划分用量、Codex 额度和 Claude Code 额度，避免形成通用的卡片拼盘。

### 用量统计

沿用现有数据和周期切换：

- 今天、7 天、30 天、全部。
- 会话数、消息数、Token 数和缓存率。
- Token 构成：输入、缓存输入、输出和推理输出；数值为零的类别可以隐藏。
- 按 Agent 来源合并后的消息数和 Token 数拆分。
- 保留单独刷新用量的能力和当前反馈状态。

默认周期继续使用现有的“今天”，不增加新的持久化字段。来源合并继续走现有 `usageStatsDisplayRows` 规则，避免同一种 Agent 因本地和远程来源重复显示。

缓存率使用 `cachedInputTokens / (inputTokens + cachedInputTokens)`，其中 `inputTokens` 是 loader 标准化后的非缓存输入。分母为零时显示 `—`。这一口径与 Kaboo 在缺少 cache creation 字段时的输入侧命中率一致；悬停文案说明计算方式。

用量区只占顶部状态总览约三分之一。会话、消息、Token 和缓存率作为四个主要数字；下方用一条轻量分段条解释 Token 构成，再用两条紧凑明细展示各 Agent 来源的消息数与 Token 数。所有信息都来自现有统计字段，不为填充空间引入成本、代码行数或活跃时长等不可靠指标。剩余宽度分给 Codex 和 Claude Code 额度。

### Agent 额度

Codex 和 Claude Code 各占一个紧凑区域，只展示：

- 5 小时剩余百分比和重置时间。
- 7 天剩余百分比和重置时间。
- 额度刷新按钮。

不在工作台展示 Code Review 等次要额度。额度不可用时显示简短原因和“打开设置”操作，不用空的进度条代替错误说明。

## 会话区

### 会话顺序与数量

- 复用当前活跃会话检测，不定义第二套运行状态。
- 工作台加载并展示最多 10 条，活跃会话整体排在前面，各组内部按最后活动时间倒序。
- 最近会话排除已经出现的活跃会话；收藏和置顶不改变工作台顺序，但仍按现有规则影响完整“会话”页。
- 不再用“活跃会话”和“最近会话”两个小标题切断列表，活跃状态继续通过会话轨迹动画表达。
- 列表在卡片内部独立滚动，不让 10 条会话撑高整个工作台。
- “查看全部”切换到“会话”Tab；存在工作台搜索词时一并带入完整会话页。

### 会话列表

所有工作台会话使用同一种普通栏目行，不使用全宽数据表。每行展示：

- 会话标题。
- 项目名称与 Agent 来源组成一条副信息。
- 最后活动时间。
- 明确标为 `Resume` 的主操作。

工作台会话行采用约 46px 的紧凑高度。标题和副信息仍保持两行，时间与 `Resume` 保持独立列；按钮高度同步收紧，在固定高度的滚动区域内保持信息密度，同时不牺牲项目、来源和恢复入口的可读性。

点击会话行主体沿用当前逻辑打开详情。点击 `Resume` 阻止列表行点击事件：活跃会话优先聚焦现有终端或应用窗口，非活跃会话使用现有会话配置和恢复路由。按钮执行中、成功和失败继续使用现有动作反馈，不新增另一套提示。

隐藏会话和在设置中被屏蔽的 subagent 不进入工作台。远程会话遵循当前是否可恢复的能力判断，不为不可恢复来源显示无效的 `Resume`。

## Workflow

Workflow 占中部右侧 50%，本阶段只作为后续迁移的稳定布局槽位。

- 始终显示简短空状态，说明等待处理和运行中的 Workflow 将来会出现在这里。
- 不读取 Workflow 数据，不显示示例流程，不提供“继续”“查看全部”等无效操作。
- 不引入 Workflow 类型、IPC 或适配器；等迁移范围确定后再围绕真实领域模型设计接口。
- 不混入 Runtime 或 Skill 状态，避免预留区域演变成无明确语义的杂项面板。

## 视觉系统

工作台继续使用现有设计 Token，不引入独立主题：

- 字体：`Geist Variable` 用于正文和标题，`Geist Mono Variable` 用于 Token、消息数、百分比和时间等数据。
- 亮色基础：`#ffffff` 应用背景、`#f3f4f6` 导航背景、`#f6f8fa` 次级表面、`#111827` 正文、`#0969da` 交互强调色。
- 来源强调：Codex 使用现有 `#19966f`，Claude 使用现有 `#d97928`；其他来源继续使用当前来源色。
- 暗色主题完全复用现有变量，不在工作台写死亮色值。
- 圆角继续使用现有 10px / 7px，主要靠边框、留白和文字层级组织内容，减少阴影。
- 用量区四个核心指标严格等分，数值使用等宽数字并共享基线，标签也共享基线，避免不同位数和单位造成视觉偏移。

左侧导航宽度约 84px，始终显示图标和文字，不提供折叠开关。设置固定在底部，内容区独立滚动。

工作台唯一的视觉标志是会话行标题左侧的细“会话轨迹”：颜色表示 Agent 来源，活跃会话的轨迹有低强度呼吸变化，最近会话保持静态。系统启用“减少动态效果”时关闭动画，仅保留颜色和状态文字。

## 响应式布局

应用默认窗口为 1280×820，最小窗口为 860×560。布局按现有窗口边界适配：

- 宽窗口：顶部用量、Codex 额度和 Claude Code 额度并排；中部会话与 Workflow 等宽分栏。
- 中等和最小窗口：用量占满一行，两张额度卡在下一行并排。
- 中部双栏在最小 860px 窗口继续保留；会话将项目和来源收进标题副行，Workflow 只保留简短空状态。所有省略文字通过悬停标题暴露完整内容。
- 页面允许纵向滚动，不通过压缩字号强行塞进一屏。
- 搜索控件不进入 Electron 窗口拖拽区域；索引刷新位于左侧导航，不与系统窗口按钮重叠。

## 数据加载与刷新

启动时按以下顺序工作：

1. 立即渲染应用框架和上一次已索引的本地数据。
2. 并行读取用量、额度、活跃会话和最近会话，区块独立更新；不为工作台扫描 Skills 或读取 Workflow。
3. 首屏可交互后再触发现有后台增量索引刷新。
4. 刷新期间保留旧数据，只显示区块级或全局轻量状态；不清空工作台，也不覆盖整个页面。
5. 刷新完成后只更新发生变化的区块和会话，不重置当前 Tab、搜索内容、滚动位置或已打开详情。

单一区块失败不阻止其他区块展示。索引失败通过轻量提示和左侧刷新图标提供重试；正常就绪状态不常驻展示。额度失败提供设置入口；统计没有数据时显示空状态，Workflow 保持固定占位。

## 状态与兼容性

- 当前 Tab 属于界面状态；应用启动默认进入“工作台”。
- 从工作台进入“会话”时明确写入目标筛选，普通点击“会话”Tab 则恢复用户原有会话筛选状态。
- 从搜索快捷键、`Resume` 和“查看全部”触发的导航均支持键盘操作和可见焦点。
- 已有详情面板、上下文菜单、远程上传下载、迁移、导出、收藏、隐藏和删除能力继续由“会话”页及详情承载。
- 中英文文案同步提供；产品名保持 AgentRecall。

## 验收标准

- 使用已有索引数据时，启动后 1 秒内出现可用的工作台内容。
- 后台刷新不阻塞 Tab 切换、搜索、查看详情或 `Resume`。
- 工作台会话最多加载 10 条，活跃会话优先且不与其后的最近会话重复；列表在卡片内部可滚动。
- 任一支持恢复的工作台会话都可通过一次点击 `Resume` 进入现有恢复流程。
- Workflow 区域只显示无操作的预留空状态，且不会触发数据读取。
- 首页展示现有会话数、消息数、Token、缓存率、来源拆分和四档时间周期；缓存率分母为零时不显示误导性的 `0%`。
- Codex 和 Claude Code 均展示 5 小时、7 天额度及重置时间。
- 工作台不加载或展示 Top 5 Skills；Skills 管理在独立 Tab 中保持完整可用。
- 工作台和“会话”Tab 均提供会话搜索；两者输入时不触发查询，按 Enter 才提交，快捷键仍进入完整“会话”页并聚焦。
- 在 1280×820 和 860×560 窗口、亮色和暗色主题下均无横向滚动或控件遮挡。
- 全局内容区不再为“索引已就绪”保留固定高度，会话页不再显示智能、最新和最早三个排序按钮。
- 工作台会话行紧凑但仍完整显示两级文字、时间与可用的 `Resume`；四项用量指标在不同数值宽度下保持对齐。
- 现有会话筛选、详情操作、远程同步和设置入口没有丢失。
