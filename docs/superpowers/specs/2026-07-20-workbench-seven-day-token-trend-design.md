# 工作台近 7 天 Token 趋势设计

## 背景

工作台顶部目前由用量、模型额度和一个空白预留位组成。用量卡承载的内容完整，但宽度接近顶部的一半；预留位没有实际信息，首屏空间利用率偏低。

本次用固定的近 7 天 Token 折线替换预留位，同时适当收窄用量卡。趋势用于快速判断最近一周 Token 消耗的起伏，不增加指标切换，也不改变左侧用量卡已有的周期选择和统计口径。

## 目标

- 顶部第三栏展示最近 7 个本地自然日的每日 Token 使用量。
- 趋势卡固定使用 Token，不提供会话数、消息数或其他指标切换。
- 今天始终位于最右侧并有明确高亮。
- 保留用量卡现有的会话、消息、Token、缓存率、Token 构成和来源明细，但将其宽度收敛到顶部区域的约 40%。
- 允许从某个趋势点进入会话页查看该自然日的会话。
- 继续复用现有统计刷新，不为趋势新增独立刷新按钮或重复请求。

## 数据口径

### 时间范围

- 按运行应用的本地时区计算自然日边界，不使用固定 24 小时倒推来代替自然日。
- 返回从 6 天前到今天共 7 个桶，按时间从早到晚排列；没有 Token 事件的日期也返回零值。
- 今天只统计本地零点到当前统计请求时间，未来时间戳不进入趋势。
- 每个桶携带开始时间和下一个自然日的排他结束时间，前后端都使用同一组边界，避免午夜事件重复落入两个日期。

### Token 范围

- 每日总量沿用现有 Token 事件口径，包含输入、缓存输入、输出和推理 Token。
- 远程同步得到且带有时间戳的 Token 事件与本地事件一同统计。
- 跨来源镜像事件继续按现有 `dedupeKey` 规则去重，不能因为按天分组而重复计数。
- 是否排除 Subagent 继续服从现有统计设置，与左侧用量统计一致。
- 缺少带时间戳 Token 事件的历史数据不按会话总量猜测日期，也不做平均分摊；对应日期保持真实可证明的事件总量。

## 数据模型与查询

扩展现有 `SessionStats` 响应，增加固定七项的每日 Token 数组。每项包含：

- `dayStart`：本地自然日起点时间戳。
- `dayEndExclusive`：下一个本地自然日的排他时间戳。
- 输入、缓存输入、输出、推理和总 Token。

趋势独立于当前用量周期：即使左侧选择“今天”“30 天”或“全部”，趋势仍固定为近 7 个自然日。

`SessionsStore.getStats` 在同一次调用中完成现有汇总和七日趋势查询。查询先在完整七日范围内按现有优先级去重 Token 事件，再将保留的事件放入预先计算的七个自然日区间；不能先按来源或日期分别去重。这样一个镜像事件始终只贡献一次，统计刷新也只发生一次 IPC 往返。

## 卡片布局

宽窗口下顶部仍为一行三栏：

1. 用量卡约占 40%，保留现有内容与周期下拉框。
2. 模型额度卡约占 34%，保留 Codex、Claude Code 并列和共享刷新按钮。
3. 近 7 天 Token 卡约占 26%，替换空白预留位。

趋势卡与前两张卡同高。中等宽度无法容纳三栏时，趋势卡换到下一行并占满可用宽度，不隐藏；窄窗口继续按单列顺序堆叠，避免折线和文字被压缩到不可读。

## 趋势卡内容

- 标题固定为“近 7 天 Token / Token · Last 7 days”。
- 标题区显示七日 Token 总量，不放指标下拉框和刷新按钮。
- 主体使用轻量 SVG 折线和半透明面积填充，不引入新的图表依赖。
- 横向固定七个点，对应七个自然日；纵向按七日最大值归一化，并保留上下留白。
- 不显示拥挤的纵轴刻度，只在底部显示星期或短日期；今天的点和日期使用强调色。
- 底部显示每日平均值和今日累计值，均使用现有紧凑 Token 格式化规则。
- 七天全部为零时保留日期和水平基线，并显示“近 7 天暂无 Token 用量”，不制造虚假的波动。

## 悬停与键盘交互

- 鼠标悬停或键盘聚焦数据点时显示浮层，包括完整日期、总 Token、输入、缓存输入、输出和推理 Token。
- 数据点可通过 Tab 聚焦，并提供包含日期和总量的无障碍标签。
- 浮层限制在卡片内部，靠近边缘的数据点自动调整对齐，不能被卡片裁切。
- 遵循减少动态效果设置；折线不依赖入场动画表达数据。

## 单日会话联动

- 点击或按 Enter 激活某个数据点后切换到“会话”页，并应用该点对应的单个自然日范围。
- 该入口展示的是“该日有活动的会话”，沿用会话查询现有的最近活动时间口径；它不是 Token 事件逐条归因列表。
- 因为趋势本身是全局统计，联动时清空会话关键词和项目、来源、环境、标签、Open 状态等旧筛选，恢复默认可见范围，只保留所选自然日，避免旧筛选让钻取结果与趋势卡明显不一致。
- 会话页显示一个带完整日期的临时单日筛选项；选择任意现有日期预设会替换该单日筛选，清除单日筛选则回到“全部时间”。
- 查询使用 `[dayStart, dayEndExclusive)`；传给现有包含上限的 `dateTo` 条件时转换为 `dayEndExclusive - 1ms`，避免午夜边界重叠。

## 状态与刷新

- 初次加载、用量周期变化和手动刷新继续只调用现有统计接口，响应同时更新左侧用量与七日趋势。
- 手动刷新期间继续只旋转用量卡原有刷新按钮；趋势卡不增加第二个刷新入口。
- 统计失败沿用现有反馈，不保留一份看似最新但实际已与用量卡脱节的趋势状态。

## 实现边界

- 不增加数据库字段或迁移，不改变 Token 事件写入和远程同步格式。
- 不引入图表库；折线、面积、数据点和浮层在现有工作台组件内实现，并只为独立的图表交互边界保留内部组件。
- 不移除用量卡信息，不改变模型额度的数据源、刷新逻辑或错误状态。
- 不增加趋势指标切换、周/月切换、来源拆线、预测或成本换算。
- Workflow 预留区和工作台会话列表保持不变。

## 验收标准

- 工作台第三栏不再显示空白占位，而是固定的近 7 天 Token 折线。
- 折线始终包含最近七个本地自然日，今天位于最右并高亮，空白日期显示为零。
- 七日总量和每个点的 Token 构成与去重后的 Token 事件一致，远程事件和 Subagent 设置得到正确处理。
- 左侧用量卡保留全部现有信息，宽窗口下约占顶部 40%，三张卡同高。
- 悬停和键盘聚焦可查看日期及 Token 构成；全零、单点峰值和超大数值均不溢出。
- 点击任意点可进入会话页并应用无午夜重叠的单日筛选；切换现有日期预设或清除筛选行为明确。
- 中等窗口下趋势换行而不消失，窄窗口不产生水平滚动。
- 统计存储、跨来源去重、日期边界、工作台结构和单日联动都有可观察行为测试，现有构建与测试继续通过。
