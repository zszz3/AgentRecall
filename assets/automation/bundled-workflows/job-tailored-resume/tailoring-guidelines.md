# 岗位定制规则

## 事实边界

- 简历中的每个技能、职责、项目、日期和数字都必须能追溯到主简历、工作目录证据或用户明确补充的事实。
- 可以改变顺序、压缩文字和强化表达，但不得扩大职责范围、改变任职关系或把团队成果说成个人成果。
- 没有证据的岗位要求放入匹配报告的“未覆盖项”，不得编造后写入简历。
- 不确定的内容必须删除、标记为待确认，或通过人工 Gate 询问。

## 岗位匹配

- 优先呈现 JD 最前面、重复出现或被标记为硬性要求的能力。
- 只有候选人确实具备相关事实时，才使用 JD 中的同义词和准确术语。
- 每项高优先级要求都应在匹配报告中对应至少一条证据，或明确标为未覆盖。
- 不输出“ATS 通过率”“录用概率”等无法验证的百分比分数。

## 内容写作

- 经历 bullet 优先采用“动作 + 方法/范围 + 结果”结构，使用清晰动词开头。
- 量化结果只复用来源中已有的数字；没有数字时描述可验证的范围、复杂度或交付物。
- 删除与岗位无关的重复细节，但不要删除解释关键时间线所需的信息。
- 技能列表与经历证据保持一致，避免只罗列关键词。
- 默认控制在 1 至 2 页可打印篇幅，资深候选人可保留必要的第二页。

## 格式与可解析性

- 使用常见章节名称、清晰日期、纯文本链接和单栏阅读顺序。
- 不用表格承载核心经历，不依赖图标表达联系信息或技能等级。
- HTML 与 Markdown 内容必须一致；HTML 只负责可靠排版，不增加新事实。
